Complexity Analysis

Deep Diving into Depth-First Search Performance

Time Complexity

O(V + E)

In the worst case, DFS visits every vertex (V) and explores every edge (E) exactly once. In a grid maze, this translates to exploring all reachable cells.

Space Complexity

O(V)

The space required is proportional to the maximum depth of the search tree. In a maze, the recursion stack can grow up to the total number of vertices (V) in a single long path.

How This Relates to Our DFS Maze Solver

The yellow O(n) line on the chart above represents our algorithm

⏱ TIME COMPLEXITY
O(V + E)
  • ✓ V = every cell visited once = Rows × Cols
  • ✓ E = every edge (neighbor) checked once = 4 × V max
  • ✓ Simplifies to O(n) where n = grid size
  • ✓ That's the yellow line on the chart ↑
💾 SPACE COMPLEXITY
O(V)

Why is it O(V)? Memory is used by the algorithm's recursive "call stack" to remember the path backwards to the start. In the absolute worst-case scenario (like a maze that is one single long, winding, snake-like corridor with no dead ends), DFS will travel through every single cell (V) before reaching the target.

  • ✓ In this scenario, the stack grows to hold all V cells at once.
  • ✓ Thus, peak memory usage scales exactly with the grid size O(n).
  • ✓ Keep an eye on the Max Stack (Space) metric in the demo below!

💡 Key Takeaway: DFS sits in the O(n) — Good category. It's much faster than O(n²) brute-force approaches and scales linearly with grid size. The Interactive Demo below lets you see this in action — watch how Processed (V) grows proportionally to grid size.

Formula Cheat Sheet

  • V = Number of Nodes (Cells)
  • E = Number of Edges (Neighbors)
  • Average Case: Faster path discovery
  • Memory: Depends on Stack Depth
